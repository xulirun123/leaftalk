修复了翻译键位置错误的关键问题

🚨 发现并修复的关键问题

问题：翻译键添加到了错误的位置

发现： 之前添加的翻译键被错误地放在了 "profile" 部分，而不是 "settings" 部分

影响： 导致 $t('settings.xxx') 调用时找不到对应的翻译键

修复： 将所有设置相关的翻译键移动到正确的 "settings" 部分

修复的文件结构问题

// ❌ 错误的位置（之前） "profile": { "personalInfo": "个人资料", "youthMode": "未成年模式", // ... 其他设置翻译键 } // ✅ 正确的位置（修复后） "settings": { "personalInfo": "个人资料", "youthMode": "未成年模式", // ... 其他设置翻译键 }

✅ 修复完成的语言文件

已修复的语言文件

  • ✅ 马来语 (ms.json) - 翻译键已移动到 settings 部分
  • ✅ 日语 (ja.json) - 翻译键已移动到 settings 部分
  • ✅ 韩语 (ko.json) - 翻译键位置正确
  • ✅ 繁体中文 (zh-TW.json) - 翻译键位置正确
  • ✅ 简体中文 (zh-CN.json) - 翻译键位置正确
  • ✅ 英语 (en.json) - 翻译键位置正确

🎯 修复总结

这次修复解决的根本问题

  • 🔧 翻译键位置错误：将设置翻译键从 "profile" 移动到 "settings" 部分
  • 🔧 文件结构规范：确保翻译键在正确的命名空间中
  • 🔧 重复键清理：移除了 profile 部分的重复翻译键
  • 🔧 语言一致性：所有6种语言的结构现在完全一致

技术细节：

  • Vue i18n 使用 $t('settings.xxx') 调用时，会在 messages.settings 对象中查找翻译键
  • 之前翻译键在 messages.profile 中，导致查找失败
  • 现在翻译键在正确的 messages.settings 中，查找成功
